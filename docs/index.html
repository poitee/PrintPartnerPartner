Print Partner

Self-hostable web workflow for layered STL kits — Sources through Checkoff.

What it is

Print Partner runs in your browser and helps you sync GitHub or local repos, merge base and add-on layers, assign role filament colors, export STLs by role and folder, and track what you have already printed. Deploy with Docker on one port — data stays in a local volume or directory you control. No cloud account required for self-host.

Workflow

  • Sources — add repos or local folders; categories; import rules; update badges; search STLs across repos.
  • Build — set role filament colors, attach sources to a plan, pick files, set quantities, read repo docs inline, update build.
  • Review — confirm totals by role and filament and fix sync issues before export.
  • Checkoff — mark printed units; export an HTML checklist or the missing STLs to a rebuilt stl-missing/ folder.
Sources → Build → Review → Checkoff

Sync repos → compose build → validate & export → track prints

Get started

Self-host with Docker Compose:

git clone https://github.com/poitee/PrintPartnerPartner.git
cd PrintPartnerPartner
docker compose up --build

Then open http://localhost:8080.
